# User guide

Pulpo turns experiment metadata into a **flow diagram on the Figma canvas**. Only the **experiment name** is required — everything else improves the diagram.

The plugin form is one scrollable page with **Experiment**, **Goals**, and **Journey** sections (not separate tabs).

## Topics

| Topic | On canvas | Learn more |
| --- | --- | --- |
| [Experiment details](/guide/experiment) | Overview card with title, status, dates, audience, owner | Title, status, **Details** (hypothesis, dates, owner), **Rolled out** when Concluded |
| [Goals & metrics](/guide/goals) | Primary and secondary metrics with direction and thresholds | Primary metric, direction (increase / decrease / neutral), guardrails |
| [Journey mapping](/guide/journey) | Entry, touchpoints, variant branches, exit, connector spine | Touchpoints, **Add design variant**, frame links |
| [Variants](/guide/variants) | Variant cards with thumbnails and concluded metric values | Names, colors, Figma links, rollout highlight |
| [Resources](/guide/resources) | Smart icons for Figma, Jira, Linear, Notion, GitHub, and more | Link designs, tickets, briefs |
| [Updating your flow](/guide/refresh) | Re-run **Create flow** after form edits; connectors auto-update when you move frames | When to recreate vs move cards on canvas |

## Plugin UI

- **Expand window** — toggle icon at the top of the form for a larger plugin panel
- **Cancel** — dismisses the current action with a toast; the plugin stays open
- **Contact** — intro **Contact & feedback** or form footer **Contact** → [Contact](/contact) page

## Validation

If something blocks the flow from building, Pulpo shows a **toast in the plugin** with up to five issues under **Experiment**, **Goals**, or **Journey**.
