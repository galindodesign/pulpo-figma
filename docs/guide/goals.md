# Goals & metrics

Goals define **what success looks like**. Each goal becomes a metric row on the experiment overview and in variant comparisons.

## Where to find it

**Goals** tab in the plugin.

## How to use it

1. Add one or more goals — each goal is a metric you measure
2. The **first goal is primary** and drives the outcome summary
3. Set **direction** (increase or decrease) and optional **thresholds**
4. Drag to reorder — move your primary metric to the top

## Expected result

A goals table on the experiment overview, with variant rows showing metric values when concluded.

## Tips

- Use one **primary** metric for the decision; add guardrails as secondary goals.
- Name metrics clearly — "Checkout conversion rate (CVR)" beats "Metric 1."

## Common mistakes

- Adding many goals without marking which one decides the test — keep one primary.
