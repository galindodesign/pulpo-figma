# Journey mapping

The journey shows **where the experiment runs** in the user path — entry, touchpoints, variants, and exit.

## Where to find it

**Journey** section in the plugin (scroll down from Experiment and Goals).

## How to use it

1. Review the default **Entry** and **Exit** rows
2. Add **touchpoints** with the inline add rows or **+** insert spots between steps
3. On the touchpoint where the test runs, click **Add design variant**
4. Optionally link a Figma frame to a touchpoint via its menu (⋮) for a design thumbnail on the canvas
5. Drag touchpoints to reorder the path

Entry and exit are created automatically. Pulpo infers experiment type from variant count: **2 variants = A/B**, **3+ = multivariate**.

## Expected result

A horizontal journey spine on the canvas with touchpoint cards, connector lines, and variant branches on the touchpoints where you added them.

## Tips

- Keep journeys to **3–6 touchpoints** for readability.
- Name touchpoints after user actions — "Pricing page," not "Step 2."

## Common mistakes

- Adding variants before naming touchpoints — give each step a clear label first.
- Too many touchpoints — focus on the path that matters for the test.
